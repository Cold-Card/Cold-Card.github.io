博客升级记录（2020.04.17）

发表于 2020-04-17 更新于 2020-05-08 分类于 Hexo > 升级 热度： ℃
本文字数： 54k 阅读时长 ≈ 49 分钟

前言

本博客建成于2018年（夏），当时是在我高中毕业的那个暑假，虽然对于这些代码等知识，还不了解，但凭着对于这种东西的热爱，还是废寝忘食的完成了。
但完成后的我发现，相比起写博客，我还是喜欢建设博客的过程，那种一点一点美化/优化的过程。所以建站快两年，日志还是就那几篇。。。惭愧。

更新总览

  • 更新前版本：Hexo-3.7.1 NexT-5.1.4
  • 更新后版本：Hexo-4.2.0 NexT-7.8.0

经过两年的更新，NexT主题相比原来集成了很多插件，比起当时，此版本在美化和优化设置上要简单得多。但相对的，更新起来更加的麻烦（主要是原来的设置/拓展过多），用git pull平滑更新会报错，只能手动更新，但也是个大工程。
所以，考虑到我的日志并不多，主要的美化还是在custom.swig文件中，以及我想用一下数据文件（Data File)，最后我决定直接放弃原来的版本，重新建设新版本，原来的日志手动升级过来。

数据文件

自从 NexT-7.3.0 开始，官方推荐采用数据文件将配置与主题分离，这样可以在不修改主题源码的同时完成选项配置自定义布局自定义样式，便于后续 NexT 版本更新。

Steam的小黄信与小红信

发表于 2019-04-14 更新于 2020-04-27 分类于 Steam 热度： ℃
本文字数： 1k 阅读时长 ≈ 1 分钟

前言

先前，我专门写了一篇关于Steam跨区买游的日志,在文中我较详细的介绍了Steam跨区买游的方法以及安全性，结论是跨区买Key是较为安全的，本文将详细介绍商店、礼物、Key这三种方式的跨区安全问题。

小黄信与小红信

通常Steam可能的帐号及游戏的风险基本是两种，被玩家称为小黄信与小红信。

小红信

初步了解跨区买游的有些人可能会想，能不能利用“科学上网”到低价区买一些中国区售价较高的游戏呢，这样会有什么样的风险？答案很简单，你大概率会获得G胖送出的“小红信”一张。

那小红信的作用是什么，收到“小红信”之后，如果你无法出示跨区所在国家的居住证明或来往这些地区的机票等证明，账户会被永久封停。目前在淘宝上有很多商户出售低于国区售价的游戏，并且说明需要提供账号密码进行代激活。他们的操作方式很可能是用你的账号跨区，然后用低价区货币进行结算，也可能是跨区来接受低价礼物。无论如何，这类购买行为都会招致G胖“小红信”的“青睐”。如果你爱惜自己的账号，请切记不要以身犯险。

SteamDB 指南

发表于 2019-04-14 更新于 2020-04-27 分类于 Steam 热度： ℃
本文字数： 1.1k 阅读时长 ≈ 1 分钟

2019年8月，Steam由于更改了相关政策，本文“前言”部分内容已不再准确，但并不影响SteamDB的使用方法

前言

先前，我专门写了一篇关于Steam跨区买游的日志,在文中我较详细的介绍了Steam跨区买游的方法以及安全性，结论是跨区买Key是较为安全的，那跨区买Key则通常会出现锁区现象，本篇文章将会介绍一个功能强大，对Steam跨区买Key锁区信息查询有莫大帮助的一个网站——SteamDB。

入Steam坑有段时间了，从不知道SteamDB这个网站，也忘了在那个地方了解了这个网站，摸爬滚打到现在也可以简单使用了，SteamDB其实还有很多功能可以发掘，本着授人以鱼不如授人以渔的想法写下这篇指南，希望能帮助到你们。（已经可以熟练使用的大佬可能会觉得这篇指南写的很啰嗦，纯萌新向嘛 _(:зゝ∠)_）。

SteamDB 简介

SteamDB是一个关于Steam数据库的一个网站，这个网站的建立是为了让用户更深入了解Steam数据库。它可以追溯应用（Application）和包（Package）的更新情况并将他们保存在历史中。同时也提供一些像Calculator一样的工具给用户，让用户更了解自己的Steam账户。
